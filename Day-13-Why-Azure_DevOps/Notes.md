# Day 13 - Azure DevOps

## Why DevOps Engineers use Azure DevOps?

Azure DevOps is a platform that provides a set of services to automate and manage the entire Software Development Life Cycle (SDLC).

### Key Flow:
- Azure DevOps → Platform
- Platform → Services
- Services → Automates SDLC
- Result → Reduced manual effort and faster delivery (Ship faster)

---

## SDLC (Software Development Life Cycle)

1. Planning  
2. Design  
3. Development  
4. Testing  
5. Deployment  
6. Maintenance / Monitoring  

---

## Azure DevOps Core Services Flow


### 1. Azure Boards
- Used for Planning
- Track work items (Tasks, Bugs, User Stories)

### 2. Azure Repos
- Version control system (Git)
- Stores source code
- Example: v1, v2 code versions

### 3. Azure Pipelines
- Implements CI/CD (Continuous Integration / Continuous Deployment)
- Automates:
  - Build
  - Test
  - Deploy

### 4. Test Plans
- Used for Testing strategy
- Supports manual and automated testing

### 5. Artifacts
- Stores build outputs (packages, binaries)
- Used for deployment and reuse

---

## Overall Workflow

1. Plan work using **Boards**
2. Develop code and store in **Repos**
3. Build & Deploy using **Pipelines (CI/CD)**
4. Perform testing using **Test Plans**
5. Store outputs in **Artifacts**
6. Achieve Automation across SDLC

---

## Key Benefits

- End-to-end SDLC management
- Automation reduces manual work
- Faster and reliable deployments
- Better collaboration between teams
- Continuous monitoring and improvement
